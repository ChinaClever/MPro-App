快速上手

提供两种方式让使用者快速上手，可根据需要选择合适的使用方式。

目录结构

ARM UI 提供两种资源包（详情请移步 指南 → 开始使用），解压后其目录结构分别如下：
ARM UI 生产包 (armui_dist.zip)
ARM UI 开发包 (eastups-ARM-Web-master.zip) 
- [include]  /* 包含已压缩的页面公用资源，如js、css、fonts等 */
- favicon.ico  /* 浏览器地址栏小图标 */
- index.html   /* 入口网页文件（首页） */
- inner_page.html  /* 内页示例文件 */
/* 除包含开发包中所有资源外，还包括以下内容 */
- [pages]  /* ARM UI 的文档文件 */
- [source]  /* 包含页面公用资源的源码文件 */
- gulpfile.js  /* Gulp 任务配置文件 */
- package.json  /* 依赖包配置文件 */

ARM UI 的整体页面结构由两部分组成：

  • 父级：实现导航功能，提供一个 iframe 容器，用来调用并显示子页面。即上面的 index.html 文件。
  • 子页面：或称内页，是实现软件功能的一众业务页面，根据父级的调用（点击导航）显示在父级，响应用户操作，完成各个业务功能。例如上面的 inner_page.html 文件。

而 include 文件夹下的资源，就是上述两部分中页面所调用的各种公用资源，包含CSS样式表、JS脚本、iconfont字体文件、图片等。

使用方法

ARM UI 生产包 (armui_dist.zip)
ARM UI 开发包 (eastups-ARM-Web-master.zip)
  1. 将所有文件及文件夹复制到项目中；
  2. 根据实际需要修改 index.html 中的标题名称（项目名称）；
  3. 按照 inner_page.html 中的 HTML 结构编写内页（业务页面）；
  4. 编写业务页面时，需要注意引用资源的相对路径填写正确；
  5. 如需在页面上使用某个组件，请参阅 组件 栏目下对应组件的使用文档；
  6. 业务页面编写完毕后，定位到 include → js 目录，找到 nav.json 文件，该文件是导航配置文件，请根据实际情况编写、修改。其中：
    • name：该字段设置导航项目的名称；
    • icon：该字段设置导航项目的小图标，填写图标名称即可（详情参阅 组件 → 图标）；
      注：小图标只对一级导航有效
    • url：该字段设置导航项目的链接地址，指向对应的业务页面；
    • base：该字段只对包含二级导航的一级导航有效，可设置一个基准地址，所有二级导航的 url 均按此地址作为上级地址；
    • subNav：该字段设置一级导航下的二级导航内容。
